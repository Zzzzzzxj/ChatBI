# Join 路径推导

## 背景

表级召回和字段级匹配能够识别当前问题涉及哪些表和字段，但多表查询还需要明确主表、连接路径和 ON 条件。企业数据库中不一定完整维护外键约束，仅依赖模型自行推断容易出现漏 Join、错 Join 或关联条件错误。

## 本次变更

- 新增 Join 路径推导模块，维护核心业务表之间的关系图。
- 支持根据查询意图和候选表选择 SQL 锚表。
- 使用 BFS 查找锚表到目标表的最短连接路径。
- 支持生成包含 JOIN 类型和 ON 条件的 FROM/JOIN SQL 片段。
- 对无法连通的表返回 `unreachable`，便于后续拆分为独立查询。

## 关键文件

- `join_resolver.py`

## 验证结果

- Python 模块编译检查通过。
- 单表、两表、多表和不可连通表场景验证通过。
- 敏感表述扫描通过。
