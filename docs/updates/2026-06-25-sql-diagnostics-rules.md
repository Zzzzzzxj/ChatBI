# SQL 诊断与业务规则防护

## 背景

基础 Text2SQL 链路可以运行后，仍会出现字段口径、Join 遗漏、时间边界、过滤条件和聚合维度等问题。项目需要把常见错误显式纳入系统约束和诊断能力。

## 本次变更

- 强化 Prompt 中的业务规则和错误防护约束。
- 新增 SQL 诊断模块，用规则识别字段错误、Join 问题、时间问题、过滤遗漏、聚合错误和非只读操作。
- 在主流程返回结果中增加诊断信息，方便观察模型输出质量。
- 支持通过开关控制 Few-shot、业务规则和错误防护是否启用。

## 关键文件

- `prompt_builder.py`
- `error_analyzer.py`
- `main.py`

## 验证结果

- Python 编译检查通过。
- 典型查询可生成并执行 SQL。
- 诊断模块可识别错误使用金额字段、缺少汇率关联和缺少订单过滤等问题。

## 后续衔接

诊断结果可作为后续 Prompt 调整、指标知识注入和自动评估的依据，帮助系统从可运行逐步走向可治理。
