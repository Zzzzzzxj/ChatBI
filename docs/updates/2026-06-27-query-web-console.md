# 查询工作台交互完善

## 背景

ChatBI 已具备自然语言查询 API、流式 SQL 生成和基础页面入口。为了让业务查询链路更接近可操作的产品形态，需要补齐前端查询过程中的状态反馈、结果展示和异常处理。

## 本次变更

- 将基础查询页面升级为查询工作台，支持历史查询卡片展示。
- 增加流式 SQL 光标效果，SQL 生成完成后展示复制操作。
- 增加查询取消能力，避免长时间模型调用阻塞页面。
- 展示 SQL 生成耗时、查询总耗时和结果行数。
- 优化空结果、网络错误、模型错误和 SQL 执行错误的页面反馈。
- 在 API 根路径增加前端页面入口地址，方便服务启动后快速访问。

## 关键文件

- `static/index.html`
- `api_service.py`

## 验证结果

- 静态页面可以通过 `/static/index.html` 访问。
- 流式查询接口仍保持 `/api/v1/query/stream`。
- 空白问题可返回 SSE 错误事件，不触发模型调用。
- Python 模块编译检查通过。
- 敏感表述扫描通过。

## 后续衔接

后续可以继续增强页面的结果解释能力、查询历史持久化和多轮上下文处理，让查询工作台承接更复杂的经营分析任务。
