# 项目蓝图初始化

## 背景

ChatBI 需要先明确项目定位、业务场景和最小可运行边界，避免后续实现只停留在单点 SQL 生成脚本。项目目标聚焦企业经营分析场景，让业务人员可以用自然语言查询收入、利润、费用、客户、产品等经营数据。

## 本次变更

- 建立 ChatBI 项目说明与整体蓝图。
- 明确业务痛点：业务口径、Schema 理解、多表关联、结果可信和复杂分析拆解。
- 定义 MVP 链路：用户提问、Schema 注入、Prompt 构造、LLM 生成 SQL、SQL 执行、结果返回。
- 确定后续演进方向：数据建模、Text2SQL、服务化接口、前端交互、Schema Linking、RAG 和复杂分析流程。

## 关键文件

## 验证结果

- 仓库以项目蓝图作为初始内容，未包含业务实现代码。
- 文档只描述项目目标、架构边界和后续演进方向。

## 后续衔接

后续更新应围绕蓝图逐步补齐数据环境、查询链路、评估体系、服务接口和业务知识能力，保证仓库历史呈现真实项目演进。
