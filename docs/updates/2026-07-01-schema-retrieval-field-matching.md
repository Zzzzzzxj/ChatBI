# Schema 召回与字段匹配

## 背景

随着业务表和字段数量增加，直接向模型注入全量 Schema 会带来上下文冗余、字段选择不稳定和业务口径混淆的问题。系统需要先判断当前问题涉及哪些表，再进一步识别相关字段和强制业务口径。

## 本次变更

- 新增表级召回模块，使用 LangChain Embeddings 与 ChromaDB 持久化向量库管理表描述索引。
- 为客户、产品、销售订单、汇率和费用等核心业务表维护结构化描述、领域类型和关键字段。
- 新增字段级匹配模块，为业务字段维护语义描述，并在候选表范围内进行字段召回。
- 增加业务规则层，处理收入口径、含税金额、成本口径、订单状态、汇率换算和费用层级等字段选择约束。
- 增加 Embedding 模型配置项，复用项目已有 OpenAI-compatible 服务配置。
- 忽略本地 ChromaDB 持久化目录，避免将运行时索引数据提交到代码仓库。
